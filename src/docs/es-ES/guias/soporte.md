# Soporte

¿Necesitas ayuda con CubicLauncher? Disponemos de varios canales oficiales donde puedes obtener asistencia, reportar problemas y recibir ayuda tanto de la comunidad como del equipo de desarrollo.

## Antes de solicitar soporte

Antes de abrir una solicitud de soporte, asegúrate de:

- Revisar la documentación oficial.
- Verificar que estás utilizando la última versión de CubicLauncher.
- Buscar si tu problema ya ha sido reportado anteriormente.
- Reunir registros (logs), capturas de pantalla o mensajes de error relevantes.

## GitHub Issues

Nuestro método principal para reportar errores, solicitar nuevas funcionalidades o recibir soporte técnico es a través de los **Issues** del repositorio oficial.

### ¿Cuándo utilizar GitHub Issues?

Utiliza GitHub Issues para:

- Reportar errores (bugs).
- Solicitar nuevas funcionalidades.
- Informar problemas de compatibilidad.
- Reportar comportamientos inesperados.
- Realizar seguimiento de problemas conocidos.

### Repositorio Oficial

**Repositorio:** https://github.com/CubicLauncherDevs/CubicLauncher

Al crear un issue, intenta incluir:

- Versión de CubicLauncher.
- Sistema operativo utilizado.
- Descripción detallada del problema.
- Pasos para reproducir el error.
- Capturas de pantalla o registros relevantes.

## Soporte en Discord

También ofrecemos soporte a través de nuestro servidor oficial de Discord.

**Servidor de Discord:** https://discord.com/invite/7VaqSrPukm

### Canal de soporte público

Para dudas generales y problemas comunes, puedes utilizar el canal público de soporte.

Este canal es ideal para:

- Ayuda con la instalación.
- Problemas de configuración.
- Consultas generales.
- Resolución rápida de dudas.

Tanto miembros de la comunidad como miembros del equipo podrán ayudarte.

### Sistema de Tickets

Si tu problema requiere atención personalizada o involucra información privada, puedes abrir un ticket de soporte.

Recomendamos utilizar tickets para:

- Problemas complejos.
- Seguimiento individualizado.
- Casos que requieran compartir información privada.
- Soporte avanzado.

Una vez creado el ticket, un miembro del equipo revisará tu caso y te responderá lo antes posible.

## ¿Sigues necesitando ayuda?

Si no encuentras una solución en la documentación, puedes contactarnos a través de GitHub Issues, el canal público de soporte en Discord o mediante un ticket de soporte.
