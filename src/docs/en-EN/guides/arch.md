# Arch Linux

## Installation

On Arch Linux it is recommended to compile CubicLauncher locally using the official `PKGBUILD`. There is no need to clone the full repository or install dependencies manually: the `PKGBUILD` itself downloads the source code and `makepkg` handles resolving and installing the necessary dependencies.

```bash
mkdir cubiclauncher-build
cd cubiclauncher-build

wget https://raw.githubusercontent.com/CubicLauncherDevs/CubicLauncher/main/dist/arch/PKGBUILD
makepkg -si
```

## Updating to a new version

When a new version of CubicLauncher is released, you can update the locally compiled package without downloading the `PKGBUILD` from scratch.

Go to the directory where you downloaded the `PKGBUILD`, edit the file and change the `pkgver` value to the new version:

```bash
cd cubiclauncher-build
nano PKGBUILD
```

Look for the `pkgver=` line and update it. Then update the checksums automatically with `updpkgsums`:

```bash
updpkgsums
```

Finally, recompile and install the updated package:

```bash
makepkg -si
```

## Auto-update does not work on Arch

CubicLauncher includes a built-in auto-update mechanism that downloads the latest binary from GitHub Releases and replaces the current executable. On Arch Linux this mechanism **does not work** and cannot be fixed, for the following reasons:

1. **System permissions**: When installing via `makepkg -si`, the binary is copied to `/usr/bin/cubiclauncher` with **root** permissions (`root:root`). The normal user does not have write permissions over `/usr/bin`, so the auto-updater cannot overwrite the file.

2. **Binary incompatibility**: Even if the auto-updater were run as root, the precompiled binaries from GitHub Actions are linked against Ubuntu libraries, which can cause failures on Arch.

3. **Arch philosophy**: On Arch Linux the correct way to update any program is through `pacman`. Using external auto-updaters bypasses the package manager, leaving the system in an inconsistent state (pacman will not know the binary was replaced).

## Common issues

### Error downloading dependencies (404)

If during installation you see errors similar to the following:

```text
error: failed retrieving file 'gst-plugins-good-*.pkg.tar.zst'
error: failed retrieving file 'gst-plugins-bad-*.pkg.tar.zst'
error: failed to commit transaction (failed to retrieve some files)
```

It is likely that one of the mirrors configured on your system is outdated. Before running `makepkg` again, update the package database and sync the system:

```bash
sudo pacman -Syu
```

If the problem persists, update your mirror list or try newer mirrors:

```bash
sudo pacman -S reflector
sudo reflector --latest 20 --sort rate --save /etc/pacman.d/mirrorlist
sudo pacman -Syyu
```

Once the system is updated, run again:

```bash
makepkg -si
```
