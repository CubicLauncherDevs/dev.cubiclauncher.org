# Support

Besoin d'aide avec CubicLauncher ? Nous disposons de plusieurs canaux officiels où vous pouvez obtenir de l'assistance, signaler des problèmes et recevoir de l'aide de la communauté et de l'équipe de développement.

## Avant de demander du support

Avant d'ouvrir une demande de support, assurez-vous de :

- Consulter la documentation officielle.
- Vérifier que vous utilisez la dernière version de CubicLauncher.
- Rechercher si votre problème a déjà été signalé.
- Rassembler les logs, captures d'écran ou messages d'erreur pertinents.

## GitHub Issues

Notre méthode principale pour signaler des bugs, demander de nouvelles fonctionnalités ou obtenir du support technique est via les **Issues** du dépôt officiel.

### Quand utiliser GitHub Issues ?

Utilisez GitHub Issues pour :

- Signaler des bugs.
- Demander de nouvelles fonctionnalités.
- Signaler des problèmes de compatibilité.
- Signaler des comportements inattendus.
- Suivre les problèmes connus.

### Dépôt Officiel

**Dépôt :** https://github.com/CubicLauncherDevs/CubicLauncher

Lors de la création d'une issue, essayez d'inclure :

- La version de CubicLauncher.
- Le système d'exploitation utilisé.
- Une description détaillée du problème.
- Les étapes pour reproduire l'erreur.
- Des captures d'écran ou logs pertinents.

## Support Discord

Nous offrons également du support via notre serveur Discord officiel.

**Serveur Discord :** https://discord.com/invite/7VaqSrPukm

### Canal de support public

Pour les questions générales et les problèmes courants, vous pouvez utiliser le canal public de support.

Ce canal est idéal pour :

- Aide à l'installation.
- Problèmes de configuration.
- Questions générales.
- Résolution rapide de questions.

Les membres de la communauté et les membres de l'équipe peuvent vous aider.

### Système de Tickets

Si votre problème nécessite une attention personnalisée ou implique des informations privées, vous pouvez ouvrir un ticket de support.

Nous recommandons d'utiliser les tickets pour :

- Problèmes complexes.
- Suivi individualisé.
- Cas nécessitant le partage d'informations privées.
- Support avancé.

Une fois le ticket créé, un membre de l'équipe examinera votre cas et vous répondra dès que possible.

## Vous avez encore besoin d'aide ?

Si vous ne trouvez pas de solution dans la documentation, vous pouvez nous contacter via GitHub Issues, le canal public de support sur Discord ou via un ticket de support.
